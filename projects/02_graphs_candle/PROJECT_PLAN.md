# План: Trading Diary

**Создано:** 2026-04-22

---

## Концепция

Единая система для хранения, сопровождения и визуализации данных разными методами.

**Объект** (название не определено):
- Уникальный `id`
- Тип объекта: сделка, карточка, проект, и др.
- Методы визуализации: "ui карточка" (первый), графики, таблицы, и др.

---

## Задача 1: UI Карточка

**Описание:**  
Первый метод визуализации - UI карточка для объекта (сделка и др.).

### Подзадачи

- [ ] 1.1 Определить модель объекта (базовые поля: id, тип, данные)
- [ ] 1.2 Создать UI карточки (стилизация)
- [ ] 1.3 Реализовать рендер карточки по id объекта
- [ ] 1.4 Добавить эндпоинт/маршрут для карточки

### UI Карточка

- Показывает данные объекта
- Содержимое зависит от типа объекта
- Интерактивность: клик, hover, кнопки действий

## Задача 2: Отображение сделок на графике

**Описание:**  
Есть данные по сделкам. Нужно отобразить график от начала данных. На каждую сделку — карточка.

### Подзадачи

- [ ] 2.1 Определить формат данных сделок (входные данные)
- [ ] 2.2 Создать SVG график с маркерами сделок
- [ ] 2.3 Привязать карточку к маркеру сделки
