# Стратегия «База MA»

Moving Average Backtest & Signal Engine — модуль глубокого анализа скользящих средних для трекера сделок Bitget.

---

## 1. Типы скользящих средних (8 типов)

| # | Аббр. | Полное имя | Формула | Особенность |
|---|-------|-----------|---------|-------------|
| 1 | SMA | Simple Moving Average | `sum(P, N) / N` | Равный вес всем ценам |
| 2 | EMA | Exponential Moving Average | `P_t × k + EMA_{t-1} × (1-k)`, k=2/(N+1) | Больший вес последним ценам |
| 3 | WMA | Weighted Moving Average | `sum(P_i × i) / sum(i)` | Линейно убывающий вес |
| 4 | HMA | Hull Moving Average | `WMA(2×WMA(P,N/2)−WMA(P,N), sqrt(N))` | Минимальный лаг |
| 5 | DEMA | Double Exponential MA | `2×EMA(P,N) − EMA(EMA(P,N),N)` | Двойное сглаживание |
| 6 | TEMA | Triple Exponential MA | `3×EMA − 3×EMA(EMA) + EMA(EMA(EMA))` | Тройное сглаживание |
| 7 | KAMA | Kaufman Adaptive MA | `P×ER + KAMA_{t-1}×(1-ER)` | Адаптация к волатильности |
| 8 | ZLEMA | Zero-Lag Exponential MA | `EMA(P_shifted, N)`, shift=(N-1)/2 | Сдвиг для устранения лага |

Все реализации в `infographics/trade_analyzer.py`, метод `compute_ma(type, period)`.

## 2. Стратегии (блоки A-I)

### Block A: Цена пересекает одну MA

**Правило:** вход в лонг при закрытии выше MA, выход при закрытии ниже.

```
long_signal = close[i] > ma[i]
exit_signal = close[i] < ma[i]
```

**Тест:** 8 типов MA × 23 периода [5, 10, 15, 20, 25, 30, 34, 40, 50, 55, 75, 89, 100, 125, 150, 175, 200, 210, 233, 250, 270, 289, 300] = 184 комбинации.

**Метрики:** CAGR, MaxDD, Sharpe, сделки.

---

### Block B: Кроссовер двух MA

**Правило:** вход в лонг при пересечении быстрой MA через медленную снизу вверх, выход при обратном пересечении.

```
long_signal = fast_ma[i] > slow_ma[i] and fast_ma[i-1] <= slow_ma[i-1]
exit_signal = fast_ma[i] < slow_ma[i] and fast_ma[i-1] >= slow_ma[i-1]
```

**Тест:** 14 быстрых периодов × 17 медленных × 3 типа MA (SMA, EMA, WMA) = 714 комбинаций.

**Быстрые периоды:** [5, 8, 9, 10, 13, 20, 21, 30, 34, 40, 50, 55, 75, 89]

**Медленные периоды:** [20, 25, 30, 34, 40, 50, 55, 60, 75, 100, 125, 150, 175, 200, 210, 233, 250, 270, 289, 300]

---

### Block C: Буферная зона и подтверждение

**Буфер (buffer):** вход только когда цена выше MA + X%, выход при цене ниже MA - X%.
```
entry = close[i] > ma[i] * (1 + buffer_pct)
exit  = close[i] < ma[i] * (1 - buffer_pct)
```

**Подтверждение (confirmation):** вход после N дней подряд закрытия выше MA.
```
entry = all(close[i-n:i] > ma[i-n:i])
exit  = all(close[i-n:i] < ma[i-n:i])
```

**Варианты:** буфер ±1%, ±3%, ±5%. Подтверждение 3д, 5д.

---

### Block D: Тройная MA

**Правило:** лонг когда быстрая > средняя > медленная. Выход при нарушении порядка.

```
long_signal = fast_ma[i] > mid_ma[i] > slow_ma[i]
exit_signal = fast_ma[i] < mid_ma[i] or mid_ma[i] < slow_ma[i]
```

**Комбинации:** (5/20/50), (10/20/50), (10/50/200), (20/50/200), (13/34/89 — Fibonacci EMA), (9/21/55 — EMA), (7/14/28), (10/30/100), (20/100/250), (50/100/200).

---

### Block E: MA + дополнительные фильтры

К базовому кроссоверу добавляются фильтры:

| Фильтр | Условие входа |
|--------|--------------|
| RSI > 50 | `rsi[i] > 50` |
| Наклон MA > 0 | `slow_ma[i] > slow_ma[i-N]` |
| Объём > среднего | `volume[i] > SMA(volume, 20)[i]` |

Сигнал активируется только если базовое условие **И** фильтр совпадают.

---

### Block F: MA как динамический стоп-лосс

Вход по кроссоверу. Выход при закрытии ниже короткой MA (трейлинг-стоп).

**Стоп-периоды:** SMA 10, 15, 20, 30, 50.
**Базовый кроссовер:** SMA 20/50, SMA 50/200.

---

### Block G: Золотой крест и крест смерти

**Золотой крест:** SMA 50 пересекает SMA 200 снизу вверх.
**Крест смерти:** SMA 50 пересекает SMA 200 сверху вниз.

Вариации:
- Классический: вход на золотом кресте, выход на кресте смерти
- Контрарный: вход на кресте смерти, выход через N дней
- С задержкой: вход через N дней после креста

---

### Block H: Сравнение рынков

Прогон топ-5 кроссоверов на подмножествах:

## 3. Метрики производительности

| Метрика | Формула | Описание |
|---------|---------|----------|
| **CAGR** | `(equity_end / equity_start)^(1/years) - 1` | Среднегодовая доходность |
| **MaxDD** | `min((equity_peak - equity_trough) / equity_peak)` | Максимальная просадка |
| **Sharpe** | `mean(returns) / std(returns) × sqrt(252)` | Доходность на единицу риска |
| **Win Rate** | `wins / total_trades` | Доля прибыльных сделок |
| **Trades** | кол-во | Число сделок за период |
| **Exposure** | `days_in_market / total_days` | Время в рынке |
| **CAGR/MaxDD** | доходность к просадке | Эффективность риска |

## 4. Генерация сигналов

`ma_signal_generator()` — объединяет все стратегии и выдаёт единый сигнал:

```python
{
    'symbol': 'BTCUSDT',
    'timestamp': ...,
    'signals': [
        {'type': 'trend', 'direction': 'bullish', 'source': 'price_above_sma200', 'confidence': 85},
        {'type': 'crossover', 'direction': 'bullish', 'source': 'ema13_sma50_cross', 'confidence': 72},
        {'type': 'filter', 'passed': True, 'source': 'rsi_above_50', 'confidence': 60},
    ],
    'composite': 'bullish',
    'overall_confidence': 75,
    'recommendation': 'hold_long'
}
```

Уровни уверенности:
- 0-30: слабый сигнал, требуется доп. подтверждение
- 31-60: умеренный, можно учитывать
- 61-85: сильный, высокая надёжность
- 86-100: экстремальный, редкое совпадение всех фильтров

---

## 5. Фильтры подтверждения (дополнительные критерии)

Каждый сигнал проходит через систему фильтров:

| # | Фильтр | Параметр | Описание |
|---|--------|----------|----------|
| 1 | Буферная зона | buffer_pct | Цена отстоит от MA не менее чем на X% |
| 2 | Подтверждение | confirm_days | N дней подряд в нужной зоне |
| 3 | RSI | rsi_threshold | RSI > 50 для лонга, < 50 для шорта |
| 4 | Наклон MA | slope_days | MA растёт/падает за N дней |
| 5 | Объём | vol_multiplier | Объём выше среднего за 20 дней в X раз |
| 6 | ATR-фильтр | atr_multiplier | Волатильность не ниже X% от ATR |

Сигнал считается подтверждённым если прошёл ≥ 3 из 6 фильтров.
