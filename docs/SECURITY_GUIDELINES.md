# Security Guidelines

## 🔐 Правила хранения секретов

1. **Токены только в `.env`** — никогда в `.md`, `.py`, `.sh`, `.json`.
2. `.env` в `.gitignore` — всегда.
3. Не коммить `.env.example` с реальными значениями.
4. Использовать `source_env.sh` для загрузки переменных.

## 🚨 Что делать при утечке

1. Немедленно отозвать токен (GitHub → Settings → Developer settings → Personal access tokens).
2. Сгенерировать новый.
3. Обновить `.env`.
4. Запустить `git-filter-repo` для очистки истории.
5. `git push --force` во все remote.

## 🛡️ Профилактика

- Периодически проверять: `git log -p -S "ghp_"` (поиск токенов в истории).
- Использовать `git-secrets` или `truffleHog` для CI.
- Минимальные права у токенов (scope: только необходимое).
