## 3. Глубина данных (максимальный полезный range)

TradingView widget embed не имеет жёсткого лимита по range, но **рендерит ограниченное количество свечей** (~5000-7000 на малых TF, больше на крупных). Указывать range больше этого порога бессмысленно — визуально график не изменится.

| TF | ~макс. свечей | Мин. range | Макс. полезный range | Примечание |
|----|--------------|-----------|---------------------|-----------|
| 1m | ~5 000 | `1M` | `2M`-`3M` | Дальше размер графика не растёт |
| 3m | ~5 000 | `1M` | `3M`-`6M` | |
| 5m | ~5 000 | `1M` | `3M`-`6M` | |
| 15m | ~5 000 | `1M` | `6M`-`12M` | |
| 30m | ~5 000 | `1M` | `12M`-`18M` | |
| 1h | ~5 000 | `1M` | `12M`-`18M` | |
| 2h | ~5 000 | `1M` | `18M` | |
| 4h | ~5 000 | `1M` | `18M`-`30M` | |
| 1d | ~2 500 | `1M` | `48M`+ ... `all` | Данные реально расширяются |
| 1w | ~260 | `1M` | `60M`+ ... `all` | Данные реально расширяются |

**Практические рекомендации:**
- Для `1m`..`4h` нет смысла писать range больше `18M` — всё равно покажет одно и то же
- Для `1d` и `1w` можно смело указывать `all` — данные будут расширяться до максимума
- Дефолтный range (`1y`) через RANGE_MAP преобразуется в `12M`

## 4. RANGE_MAP (применяется в коде)

```python
RANGE_MAP = {
    "1m": "1M", "3m": "3M", "5d": "5D", "1w": "7D",
    "6m": "6M", "12m": "12M",
    "1y": "12M", "2y": "24M", "5y": "60M", "10y": "120M",
    "all": "all", "ytd": "YTD",
}
```

Неизвестные ключи передаются как есть (raw).

---

## 5. ❌ Запрещённые/неработающие TV-значения

Эти значения widget embed **не понимает** — покажет пустой график или "Something went wrong":

| Значение | Ошибка |
|----------|--------|
| `1W`, `4W`, `52W` (week suffix) | ❌ не поддерживается widget |
| `1Y`, `2Y`, `5Y`, `10Y` (year suffix) | ❌ только `{N}M` |
| `all` | ✅ (единственный текстовый) |

---

## 6. Формат range-значений в TradingView widget

```
{N}M  — N месяцев  (1M, 2M, 3M ... 240M)  ✅
{N}D  — N дней     (5D, 7D, 14D ... 365D) ✅
all   — все данные                           ✅
YTD   — с начала года                        ✅

{N}W  — N недель                             ❌
{N}Y  — N лет                                ❌
```

---

## 7. Примеры

```bash
# 1h, 2 месяца — полезно (разные данные)
sc DOTUSDT 1h 2m

# 1m, 12 месяцев — бесполезно (те же ~5K свечей что и в 3m)
sc DOTUSDT 1m 12M

# 1d, all — максимальная глубина
sc DOTUSDT 1d all

# 1w, 2y — 2 года недельных свечей
wg DOTUSDT 1w 2y
```
