Ниже план и конкретные изменения, которые я сделал, чтобы реализовать простой конвейер и кнопку в OpenCode для транскрибации YouTube.

Что я сделал и где лежат файлы
- End-to-end конвейер: загрузка транскрипта YouTube → конвертация в Markdown → сохранение в локальный кэш и суммаризация.
- Репозиторий Python-скрипта конвейера:
  - workspace/workspace/scripts/transcript_pipeline.py
- Вариант вызова из OpenCode через скрипт-подложку:
- /home/user_aioc/.config/opencode/tools/transcript.sh
  - вызывает Python-скрипт transcript_pipeline.py для заданного URL
- Весь результат сохраняется в кэш: ~/.cache/opencode/transcripts/
- Кликовая команда в OpenCode
- Новая кнопка/инструмент в TUI: простой документ-описание навыка
  - .config/opencode/skills/transcript-open.md
- Быстрый визуальный ориентир для добавления команды в OpenCode UI (пользовательская команда Transcript)
- Описание в OpenCode (для понятности и будущей интеграции в UI)
- Шаблон (в skills) можно использовать для внедрения в кнопки/меню OpenCode как локальный инструмент
- Дополнительная попытка автоматизации на YouTube URL
- Пробовал загрузку EN субтитров через yt-dlp напрямую и через скрипты-конвертеры; в некоторых случаях видео не содержит субтитров или доступ заблокирован, поэтому поведение может варьироваться.

Как работает конвейер по шагам
1) Ввод URL YouTube
   - Пример: https://m.youtube.com/watch?v=1NQ5U9CHL-4
2) Загрузка авто-субтитров ( yt-dlp )
   - Я настраиваю langs, по умолчанию en
   - Результатом являются файлы VTT: /tmp/yttranscripts/{vid}.{lang}.vtt
3) Конвертация VTT в текст
   - Убираем временные теги, подписи и прочие служебные строки
   - Получаем чистый текст транскрипта
4) Формирование Markdown
   - Front matter с метаданными
   - Раздел Transcript с текстом транскрипта
5) Сохранение и кэширование
   - Файл: ~/.cache/opencode/transcripts/{vid}.md
6) Резюмирование (опционально)
   - Если указан OPENAI_API_KEY (OPENAI_API_KEY) — отправлю текст на резюмирование в GPT-4o
   - Если ключа нет — применю простой эвристический резюмирующий блок
7) Вывод результата
   - Путь к сохранённому Markdown-файлу выводится в консоль

Как использовать в OpenCode
- Вызов через готовую команду Transcript
- wrapper-скрипт:
  - transcript https://m.youtube.com/watch?v=1NQ5U9CHL-4 en,ru
- Результат: путь к сохранённому MD-файлу в формате Markdown в каталоге ~/.cache/opencode/transcripts
- Встроенная суммаризация
- Если у вас есть OPENAI_API_KEY (через окружение OPENAI_API_KEY), конвейер выполнит резюмирование и добавит раздел Summary в конце Markdown.
- Без ключа — будет краткое резюме по умолчанию.
